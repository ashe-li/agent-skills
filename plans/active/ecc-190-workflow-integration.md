## Architecture Notes

**Architect 重點回饋：**

1. **版本標註去除** — `/design` Step 1 的 inline 版本標註（`v1.8 新增的...`）改為乾淨列舉，版本歷史由 CHANGELOG 承載
2. **Routing table 分層** — `/assist` Step 3 routing 表已有 14 行，新增 6 行需分為 "quick command dispatch" 和 "agent pipeline" 兩區塊，避免 LLM routing 飽和
3. **Log-to-file > frontmatter mutation** — defer reason 用 append-only `DEFER_LOG.md` 記錄，不觸碰 SKILL.md 檔案內容
4. **資源驗證前置** — 所有 SKILL.md 編輯前先確認 ECC 資源實際存在，避免 phantom routing entries

## Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|------------|
| Phantom resource references | HIGH | Phase 0 前置驗證，移除不存在的資源 |
| /assist routing table 飽和 (18 rows) | MEDIUM | 分層：command dispatch vs agent pipeline |
| bash script regression | LOW | 改用 append-only log，不修改現有 mv 邏輯 |
| Token overhead | LOW | 每個新 table row ~30-50 tokens，總增量 ~400 tokens |
