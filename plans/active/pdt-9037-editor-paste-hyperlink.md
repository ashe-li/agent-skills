## Overview

vocus 編輯器在複製貼上時丟失超連結格式（`<a href>`），症狀初次出現於 2026-03-12（Google Docs 來源），1 月起 Word 也有同樣問題。本計畫分三個 Phase：先從瀏覽器 DevTools 確認根因（Clipboard 格式層 vs. paste handler 層 vs. HTML 轉換層），再根據確認的根因實作最小範圍的修復，最後透過 unit test + 手動測試矩陣驗收，並由 `security-reviewer` 做 XSS 專項審查。

## Industry & Standards Reference

| 技術決策 | 參照依據 | 類型 |
|----------|---------|------|
| 讀取 `text/html` clipboard 格式以保留超連結 | W3C Clipboard API & Events — `ClipboardEvent.clipboardData.getData('text/html')` | W3C 標準 |
| Google Docs clipboard HTML 保留完整 `<a href>`，但外包大量 `<b>`/`<span>` inline style | Google Docs 實際 clipboard 輸出觀察 | 業界觀察 |
| Word clipboard HTML 帶 `mso-*` style 和 `<o:p>` 標籤，但 `<a href>` 仍存在 | Microsoft Office WordprocessingML clipboard 格式 | 業界觀察 |
| TipTap/ProseMirror：`transformPastedHTML(html)` + `addPasteRules` | TipTap v2 官方文件 | 框架官方文件 |
| Slate.js：`onPaste` → `deserialize` HTML via `slate-hyperscript` | Slate 官方範例 | 框架官方文件 |
| Draft.js：`handlePastedText` + `convertFromHTML` | Draft.js 官方 API | 框架官方文件 |
| Quill：`clipboard.addMatcher` | Quill 官方 clipboard 模組文件 | 框架官方文件 |
| XSS 清理：DOMPurify `ALLOWED_ATTR: ['href', 'target', 'rel']` | OWASP XSS Prevention Cheat Sheet | OWASP 標準 |
| `href` 協議白名單（`https?:` / `mailto:`），拒絕 `javascript:` / `data:` | OWASP XSS Prevention Cheat Sheet | OWASP 標準 |
| 外部連結強制注入 `rel="noopener noreferrer"` | OWASP：防止 tab-napping 攻擊 | OWASP 標準 |

## Implementation Steps

### Phase 1: 調查根因（在 vocus 專案執行，不修改任何程式碼）

目標：確認超連結在哪一層被丟失，縮小修復範圍。預估耗時 2-4 小時，本身即是 P1 triage 的交付物。

- [ ] **Step 1: 確認編輯器框架**
  - File: vocus 前端 repo `package.json`
  - Action: `grep -E "tiptap|prosemirror|slate|draft-js|quill" package.json`，記錄框架名稱與版本號
  - Why: 不同框架的 paste hook 位置完全不同
  - Dependencies: 無
  - Risk: 低

- [ ] **Step 2: DevTools 確認 Clipboard 格式層**
  - File: 瀏覽器 DevTools Console
  - Action: 在 vocus 編輯器頁面的 Console 貼入以下 snippet，然後從 Google Docs 複製有超連結的文字貼入，觀察輸出：
    ```js
    document.addEventListener('paste', (e) => {
      const dt = e.clipboardData;
      ['text/html', 'text/plain', 'text/rtf'].forEach(t =>
        console.log(`[${t}]`, dt.getData(t).slice(0, 600))
      );
    }, { once: true });
    ```
    在 Chrome、Safari、Firefox 各跑一次。記錄 `text/html` 是否含有 `<a href>`
  - Why: 區分「Clipboard 層已無 `<a>`」vs.「有 `<a>` 但貼入後消失」
  - Dependencies: 無（可與 Step 1 並行）
  - Risk: 低

- [ ] **Step 3: 定位 paste event handler**
  - File: vocus 前端 repo `src/`
  - Action: `grep -rn "onPaste\|handlePaste\|transformPastedHTML\|addPasteRules\|clipboardData\|clipboard\.addMatcher\|handlePastedText" src/`
  - Why: 確認修復目標檔案
  - Dependencies: Step 1
  - Risk: 低

- [ ] **Step 4: 追蹤 HTML-to-model 轉換路徑**
  - File: Step 3 找到的 handler 檔案
  - Action: 閱讀 handler，判斷 (a) 是否呼叫 DOMParser/DOMPurify/自訂 deserializer (b) `<a href>` 在哪一行被過濾或未被對應
  - Why: 鎖定「丟失超連結」的確切位置
  - Dependencies: Steps 2-3
  - Risk: 中（需讀懂框架內部 parse 邏輯）

- [ ] **Step 5: 記錄調查結論至 PDT-9037**
  - File: Notion ticket PDT-9037 留言
  - Action: 記錄 (a) Clipboard 層是否保留 `<a href>` (b) paste handler 路徑與行號 (c) 丟失 href 的確切原因 (d) 各瀏覽器差異 (e) 預估修復工時
  - Why: P1 triage 的交付物
  - Dependencies: Steps 2-4
  - Risk: 低

### Phase 2: 實作修復（依 Phase 1 確認的框架選擇路徑）

目標：最小範圍修復，只影響「`<a>` 標籤的 paste 解析」，不動其他 node/mark 的解析邏輯。

依 Phase 1 Step 1 確認的框架選擇 A/B/C/D 路徑，其餘路徑跳過。

#### 路徑 A：TipTap / ProseMirror

- [ ] **A-1: 先寫失敗的測試 — 委派 `tdd-guide`**
  - File: `src/editor/__tests__/paste-hyperlink.test.ts`
  - Action: 撰寫 5 個測試情境（Red 階段）：
    1. 貼入 `<a href="https://example.com">text</a>` → editor model 含 link mark
    2. 貼入 Google Docs 格式 HTML（`<b><span><a href>` 巢狀）→ link 保留
    3. 貼入 Word 格式 HTML（`mso-*` style + `<a href>`）→ link 保留
    4. 貼入純文字 → 不產生 link mark
    5. 貼入含 `href="javascript:alert(1)"` → link mark 不建立（XSS 防護）
  - Dependencies: Phase 1 Step 3

- [ ] **A-2: 修復 TipTap Link extension 或 `transformPastedHTML`**
  - 依調查結果擇一：
    - 若 `transformPastedHTML` 過濾了 `<a>`：換用 DOMPurify，`ALLOWED_TAGS` 含 `'a'`，`ALLOWED_ATTR` 含 `['href', 'target', 'rel']`
    - 若 Link extension 未開啟 paste 識別：確認 `autolink: true` 或覆寫 `addPasteRules()`
    - 若自訂 `parseHTML()` 缺 `<a>` 規則：加入 `{ tag: 'a[href]', getAttrs: dom => ({ href: dom.getAttribute('href') }) }`
  - Dependencies: A-1、Phase 1 Step 4

#### 路徑 B：Slate.js

- [ ] **B-1: 先寫失敗的測試** — 同 A-1 的五個情境
- [ ] **B-2: 修復 `onPaste` handler 的 HTML deserializer**
  - 加入 `case 'A': return jsx('element', { type: 'link', url: el.getAttribute('href') }, children)`
  - 確認以 `'text/html'` 取得 clipboard 資料而非 fallback 到純文字

#### 路徑 C：Draft.js

- [ ] **C-1: 先寫失敗的測試** — 同 A-1 的五個情境
- [ ] **C-2: 修復 `handlePastedText` 或 `convertFromHTML`**
  - 確認 `htmlToEntity` mapping 含 link entity

#### 路徑 D：Quill

- [ ] **D-1: 先寫失敗的測試** — 同 A-1 的五個情境
- [ ] **D-2: 修復 `clipboard.addMatcher`**
  - 加入 `'A'` tag matcher

#### 所有路徑共用（必做）

- [ ] **Step 3: XSS 清理加固**
  - Action: 在 href 進入 editor model 之前加入協議白名單過濾：
    ```js
    const SAFE_HREF = /^(https?:|mailto:)/i;
    const safeHref = SAFE_HREF.test(href) ? href : null;
    ```
    外部連結強制加 `rel="noopener noreferrer"`
  - Why: 修復 href 保留的同時必須防止 XSS，兩者必須在同一 commit 處理
  - Dependencies: 框架對應的 Step 2
  - Risk: 高（不做會有 XSS 漏洞）

- [ ] **Step 4: 執行測試確認 Green**
  - Action: `npm test -- --testPathPattern=paste-hyperlink`
  - Dependencies: Steps 2-3

## Architecture Notes

- **不需要新增檔案**：修復應侷限在現有 paste handler 或 Link extension 設定內。若需新增 sanitizer helper，建議放 `src/editor/utils/sanitize-href.ts`，函式 <30 行
- **向後相容性**：修復只影響「貼入 HTML 中的 `<a>` 標籤解析」。現有 link 建立（toolbar）、刪除、點擊等功能不受影響
- **跨瀏覽器注意**：Safari 對 `clipboardData.getData('text/html')` 在某些版本有差異。Phase 1 Step 2 應包含 Safari 測試
- **Word HTML 清理**：Word 輸出 HTML 含大量 `mso-*` style、`<o:p>` 標籤和 conditional comments。DOMPurify `FORBID_TAGS: ['o:p']` + `FORBID_ATTR: ['style']` 可移除雜訊，但 `<a href>` 應在白名單被保留
- **文件影響**：無需新增/更新文件（bug fix 不改 API 介面）

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| 根因在後端（server-side sanitization 在儲存時丟失 href） | Phase 1 Step 2 Clipboard 調查可區分。若確認後端問題，另開票追蹤 |
| XSS via pasted HTML | Phase 2 Step 3（清理加固）+ Phase 3 Step 2（security-reviewer），兩者均為阻擋性步驟 |
| 修復 Google Docs paste 破壞 Word paste（或反之） | TDD 情境 2+3 覆蓋兩種 HTML 格式；手動矩陣同時測兩種來源 |
| 不同瀏覽器 Clipboard API 差異（尤其 Safari） | Phase 1 Step 2 在 Chrome/Safari/Firefox 各跑一次 |
| Sprint 113 時間不足（到 3/22） | Phase 1 Steps 1-4 是獨立可交付的 triage 結果；Phase 2-3 可排入 Sprint 114 |
