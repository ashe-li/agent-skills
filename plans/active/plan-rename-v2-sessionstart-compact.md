## Architecture Notes

### Architect 審查結論：方案 C（v2 + 兜底）

**核心決策**：拆分 Stop hook 雙重職責，但保留最小化 Phase 2 作為兜底。

```
v1 架構：
  PreToolUse ExitPlanMode → hook.sh（寫 pending sidecar）
  Stop hook → guard.sh（Phase 1: 確認 + Phase 2: compaction 修復）

v2 架構（方案 C）：
  PreToolUse ExitPlanMode → hook.sh（不變）
  Stop hook → guard.sh（Phase 1: 確認 + Phase 2: 最小兜底 10 行掃描）
  SessionStart compact → compact-title.sh（新增，主要 compaction 修復）
```

**明確不做的事情**：
- 不在 SessionStart compact stdout 輸出 title reminder（避免 bug #15174）
- 不在 bash 快速退出路徑使用 Python（grep 更快且夠用）
- 不移除 Stop hook Phase 2（保留防禦性兜底）

**效能改善**：

| 情境 | v1 | v2 |
|------|----|----|
| 無 sidecar 快速退出 | ~30ms（bash+jq） | ~5ms（bash+grep） |
| Stop hook 有 sidecar | ~100-150ms（Python 50 行掃描） | ~80-100ms（Python 10 行掃描） |
| Compaction 修復 | 每次 response 掃描 50 行 | 只在 compaction 時觸發 |

## Risks & Mitigations

| 風險 | 機率 | 影響 | 緩解 |
|------|------|------|------|
| SessionStart compact 的 `transcript_path` 非預期值 | 中 | 高 — compact-title.sh 寫入錯誤位置 | Phase 0 先驗證 schema；失敗則退回方案 A |
| SessionStart compact hook 不觸發 | 低 | 中 — title 在 compaction 後遺失 | Stop hook Phase 2 兜底仍在 |
| grep JSON 解析遇到巢狀 JSON | 低 | 低 — session_id/transcript_path 格式固定 | 取 `head -1` 只讀第一個 match |
| sidecar 清理誤刪 pending 檔 | 低 | 高 — 正在等待確認的 title 被刪除 | 明確 skip `_pending_*` 前綴 |
| 使用者未更新 settings.json | 中 | 低 — 退回 v1 行為（guard.sh 兜底） | SKILL.md 新增升級指引 |
